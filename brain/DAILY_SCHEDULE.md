# DAILY SCHEDULE — Maven CMO Rhythm

> The daily cadence Maven runs when CC is actively working with the marketing engine. Designed around CC's life context (per [[USER]]) and [[binet-field-long-short]]'s brand:activation rhythm.
>
> Not rigid — adapts to CC's energy, Aura's pulse (when live), and active campaigns.

---

## Morning Block (6:00-11:00 local)

**CC's high-energy creative window.** Maven primes for work, doesn't demand it.

- **06:00-07:00** — Maven reads pulses (Bravo, Atlas, Aura). Surfaces any blockers in [[DASHBOARD]]. Drafts morning brief to CC if blockers exist.
- **07:00-09:00** — CC's content shoot window (when batch-shooting). Maven stands by with prompts, teleprompter-style hooks, reference structures.
- **09:00-11:00** — CC's strategy + deep-work window. Maven delivers: campaign briefs, positioning drafts, research findings. Canon-cited.

**Do:** Send long-form research briefs in morning. Creative review and approval.
**Don't:** Push operational / tactical requests that fragment creative time.

---

## Midday Block (11:00-14:00)

**Sales + client execution window.** Marketing engine runs background tasks.

- **11:00-12:00** — Discovery calls, sales conversations. Maven has pre-call teaser emails ready (see [[oasis-ai]] funnel). Content_ids for any mentioned assets pre-logged.
- **12:00-13:00** — Client delivery / Bravo coordination. Maven passive. Monitoring only.
- **13:00-14:00** — Quick-turn creative reviews. CC can approve/reject drafts in <30 min.

**Do:** Queue up approval requests here. Fast decisions.
**Don't:** Schedule strategy work. CC's deep-think bandwidth is lower post-calls.

---

## Afternoon Block (14:00-18:00)

**Execution + iteration window.** Maven's peak output time.

- **14:00-16:00** — Ad creative production, copy drafting, content registry updates, vertical pack work, canon file expansion.
- **16:00-17:00** — Platform posting (via [[late-mcp]] when live). Scheduled content goes out. Attribution event capture runs.
- **17:00-18:00** — End-of-day sync prep. [[STATE]] update. [[ACTIVE_TASKS]] reconciliation. [[SESSION_LOG]] append.

**Do:** Heavy Maven work without CC interruption.
**Don't:** Ask for CC voice-edits in this block if mornings are still open tomorrow.

---

## Evening Block (18:00-22:00)

**CC's wind-down. Brand protection mode.**

- **18:00-20:00** — Light monitoring. Attribution touches capture. Social-listening (if active). Competitive-intel scraping runs.
- **20:00-22:00** — Maven quiet. No feature requests, no creative asks.

**Do:** Batch any findings for next morning's brief.
**Don't:** Push Loom-recording requests, heavy copy reviews, or ad launches.

---

## Night Block (22:00-06:00)

**Cron time. CC unavailable.** Maven runs unattended workflows.

- Scheduled content posts via Late MCP
- Attribution capture jobs (when Phase 2 live)
- Nightly Supabase trace query (skill-activation optimization)
- Competitive intel Firecrawl runs (when live)
- Risk-register auto-scans (future Heartbeat pattern)

**Do:** Run anything requiring no CC decision.
**Don't:** Auto-publish anything under CC's name. Ever.

---

## Weekly Rhythm

### Monday — Strategy + OKR Review
- Read [[OKRs]], score confidence changes from last week
- Update [[DASHBOARD]]
- Plan week's creative batches
- Brief CC on pipeline state

### Tuesday-Thursday — Execution
- Content production (creative, review, publish)
- Campaign optimization (when live)
- Vertical / canon work

### Friday — Learning + Review
- Run [[self-improvement-protocol/SKILL]] (Protocol 4 IMPROVE) against week's work
- Update [[PATTERNS]] + [[MISTAKES]] with anything new
- Prep weekend content queue

### Weekend — Light-touch
- Scheduled content posts (automated)
- Emergency-only Maven engagement
- CC's DJ gig / Nicky's Donuts windows respected — Maven quiet

## Monthly Rhythm

### 1st Monday — Pulse-wide review
- [[BENCHMARK]] dimension re-score (Execution / Attribution / Research if data accumulated)
- [[RISK_REGISTER]] audit — any score changes?
- [[PATTERNS]] promotion review — anything at PROBATIONARY for 30+ days with 3+ successful apps?

### 15th — Mid-quarter re-plan
- OKR confidence trajectory review
- Canon citation audit — sample 5 recent briefs, are citations load-bearing?
- Content registry audit — registered vs. published gap?

### Last day of month — Month-close
- Full month MRR attribution report (when data available)
- SESSION_LOG rollup
- Client health review per brand

## Session-Type Rhythms

### "Quick question" session (<5 min)
- Answer in 1-5 sentences
- Canon citation if recommendation
- Update [[SESSION_LOG]] entry: question + answer

### "Build something" session (1-3 hours)
- Read relevant canon + client files FIRST
- Brief CC on approach before executing
- Run [[WRITING]] § Pre-Ship checklist if producing copy
- End with pulse update + git commit

### "Strategy review" session (2-6 hours)
- Start with [[DASHBOARD]] reading
- Load relevant canon deep-dives
- Interview-style with CC — no unilateral decisions
- Document in [[DECISIONS]] (if load-bearing) or [[PATTERNS]] (if procedural)

### "Self-improvement" session (4-8 hours)
- Run [[self-improvement-protocol/SKILL]] 4-loop
- HEAL: scan for drift
- OPTIMIZE: Supabase trace query
- DEVELOP: fill capability gaps
- IMPROVE: log learnings, promote patterns

---

## When to Deviate From This Schedule

Deviate when:
- CC is in crisis mode (hot client issue, platform ban, urgent launch) — all bets off
- Major canon insight arrives that requires immediate action (rare)
- Atlas spend gate opens unexpectedly — accelerate launch prep
- Aura signals CC energy is high/low (future)

Document deviations in [[DECISIONS]] so patterns emerge.
