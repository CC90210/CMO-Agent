## Weekly OKR Review Protocol

Every Monday, Maven:
1. Update `data/pulse/cmo_pulse.json` with confidence scores for each KR
2. Review prior week's actuals vs targets
3. Flag any KR that dropped >20 confidence points
4. Surface to CC via session note if action required

Every quarter-end:
1. Grade each objective: 0.7+ = success; 0.5-0.7 = partial; <0.5 = miss
2. Update [[BENCHMARK]] based on operational evidence accumulated
3. Write Q3 OKRs
