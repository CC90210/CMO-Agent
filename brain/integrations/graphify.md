## What it actually is

Graphify is **NOT an Obsidian plugin**. It's a **Claude Code skill** (Python CLI + `/graphify` slash command, `pip install graphifyy`) that turns any folder of files into a knowledge graph through:

```
detect → extract → build_graph → cluster (Leiden) → analyze → report → export
```

It walks code (AST via tree-sitter, 12 languages), markdown, PDFs, and images (Claude vision); builds a NetworkX graph; ranks "god nodes" + "surprising connections"; tags every edge `EXTRACTED | INFERRED | AMBIGUOUS` for hallucination auditing.

**Outputs:**
- `graph.html` (interactive vis.js)
- `obsidian/` (a real Obsidian vault you can open — backlinks come from inferred edges)
- `wiki/` (markdown articles per community)
- `GRAPH_REPORT.md`
- `graph.json`, `graph.graphml` (Gephi), `graph.cypher` (Neo4j)
- `--mcp` mode → MCP stdio server for Claude to query the graph as a tool

This is what CC means when he says "Obsidian graph" — it's not "the Obsidian app's built-in graph view," it's an Obsidian vault generated by Graphify from our markdown.

## What it does for Maven specifically

When run on `C:\Users\User\CMO-Agent\brain/` and `memory/`, Graphify:

1. **Cross-document concept linking** — finds where, e.g., "Jobs-to-be-Done" appears across `MARKETING_CANON.md`, `playbooks/youtube_video_pipeline.md`, `memory/DECISIONS.md`, `clients/oasis-ai.md`. Surfaces relationships I'd otherwise have to grep for.
2. **Obsidian vault export** — open it in Obsidian and visually navigate the connections between brand system, playbooks, sub-agent personas, client briefs, session log.
3. **Surprise-connection ranking** — flags non-obvious links (e.g., "this brand voice rule shows up in three client briefs but isn't in BRAND_SYSTEM.md — should it be canonized?").
4. **MCP query mode** — instead of grepping markdown at planning time, Maven could ask the graph: *"which playbooks touch OASIS pulse-lead-gen?"* and get a structured answer.
