AUDIT FIXES (turnkey-blocking)

  Fix 1 — Cut accuracy. Original `apply_cuts` used `-ss BEFORE -i` with
  `-c copy`, which snaps to the nearest preceding keyframe and leaves a
  frozen frame at the start of every segment. Across 30+ cuts on a 5-min
  voice video, the output looked visibly broken. Now transcodes each
  segment with libx264/aac at frame-accurate boundaries (CRF 18, veryfast)
  with `-ss AFTER -i` for slow-but-accurate seek. Concat step still
  stream-copies because all segments share codec params.

  Fix 2 — Real auto-ducking. The skill claims music is "auto-ducked under
  speech" but the original `add_music` only set static volume=-18dB and
  amix'd. That is not ducking — music sat at -18dB the whole time,
  drowning quiet speech and competing with loud. Replaced with
  sidechaincompress keyed off the speech track (threshold 0.05, ratio 8,
  attack 5ms, release 300ms): when speech is present, music ducks
  dynamically; when speech is quiet, music swells back up. Also switched
  `aloop=loop=-1:size=2e9` to `-stream_loop -1` (cleaner; no in-memory
  ringbuffer). Falls back to the original static-mix path if sidechain
  fails for any reason — better to ship music at fixed level than ship
  none.

  Fix 3 — Telegram 50MB fallback. Original code dropped to a text-only
  notification when video > 50MB, breaking the review UX (a typical 90s
  1080p clip at CRF 18 commonly hits 60-100MB). Now auto-shrinks to 720p
  (or 540p for very large files) at CRF 28 and retries sendVideo. Falls
  back to text-only ONLY if even the shrunk version is over 50MB. The
  caption notes that the phone preview is a downscaled transcode and the
  full version is on disk.

  Fix 4 — Format-spec crash. `cmd_edit` final summary used `:.1f` on
  values that could be the placeholder `'?'` string when no cuts were
  applied. Replaced with a `_fmt_secs` helper that handles missing/string
  values without raising TypeError.

  Fix 5 — `resp.json()` guard. Telegram normally returns JSON, but proxy
  failures and timeouts can return empty bodies or HTML error pages.
  Wrapped in `_safe_json()` to return `{}` instead of crashing.

KNOWN LIMITATIONS (deferred to V1.7)
  - Audio master step uses `-c:v copy`, which fails on ProRes/MOV input
    if the container can't hold the source codec. Real-world fix: probe
    codec, transcode video on incompatibility. Acceptable today because
    CC films on iPhone (h264) and uploads .mp4. (LOW)
  - `add_captions` re-runs Whisper instead of reusing the transcription
    from `detect_filler_words`. Duplicate work but adds < 30s on small
    model. (LOW)
  - `random.choice` for music track selection is non-deterministic — same
    video re-edited gets a different track each run. Documented; not yet
    fixed. (LOW)
